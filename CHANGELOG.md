# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- Optional `STRICT MATCH` and `STRICT MATCH OFF` commands for maximum reference-background adherence with mandatory non-pixel-exact disclosure.

### Changed

- Style-reference images now create dynamic Reference Style Profiles and take precedence over named Style Cards; presets remain available only as no-reference fallbacks or explicit user choices.

### Planned

- Rights-cleared real and synthetic example assets with observed cross-platform test records.
- Separately specified on-model and ghost-mannequin workflows.
- An optional, user-triggered prompt exporter with explicit portability warnings.

## [0.1.0] - 2026-07-19

### Added

- Shared Product Lock, role mapping, Safe Run, Fast Run, QA, and bounded source-first repair protocol.
- Direct-use packages for ChatGPT and Gemini plus capability-aware Claude instructions.
- Five Product Modules covering garments, fabric, earrings, bracelets, and reflective accessories.
- Four Style Cards: Sage Minimal Flat Lay, Clean White Studio, Warm Beige Editorial, and Dark Luxury Jewelry.
- Ecommerce and Social Output Profiles with background-first aspect-ratio changes and product-protection constraints.
- Python schema, reference, link, package, documentation, governance, and contract validation with GitHub Actions.
- Manual evaluation matrix separating real photographed-source evidence from supplemental synthetic cutouts.
- Professional English and Vietnamese READMEs and a concise platform-neutral quickstart.

### Known limitations

- No observed image-generation PASS is recorded in the repository at publication-candidate time; manual rows remain `NOT RUN` until rights-cleared platform runs occur.
- Generative editing cannot guarantee exact product fidelity or calibrated color.
- Platform capability and availability vary by interface, account, plan, region, and provider updates.
