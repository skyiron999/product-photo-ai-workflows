# Product Lock

Product Lock is the non-negotiable fidelity contract for every edit. The original product source image is authoritative for the product; a style-reference image is authoritative only for transferable visual treatment.

## Lock before editing

Identify the product and record every visible attribute that must survive the edit:

- geometry, scale, orientation, perspective, silhouette, fold pattern, and arrangement;
- product count and the spatial relationship between pieces;
- color relationships, tonal variation, material, weave, texture, transparency, and finish;
- construction details such as seams, hems, stitching, fasteners, clasps, settings, links, and edges;
- printed or engraved text, logos, labels, patterns, and other identity details;
- for jewelry, every visible stone, bead, charm, connector, backing, reflection, and specular highlight.

Preserve natural imperfections that belong to the photographed product. Remove only the existing background and defects that are clearly part of that background. Do not beautify, redraw, reshape, complete, simplify, duplicate, or invent any portion of the product.

## Color fidelity

Match the source photograph's observable product color and internal color relationships. Background grading must not alter product color. In every render instruction, state explicitly that background grading must not alter product color, including through reflected tint, global color casts, relighting, or contrast changes applied across the product.

Describe color using observable language when calibration data is absent. The binding rule is: do not infer exact Hex or RGB values from an ordinary photograph without calibrated user input. If exact production color is critical, request a calibrated color target or user-supplied value and mark the result for manual review.

## Reference boundary

Extract only background, lighting, shadow, spacing, palette, mood, camera treatment, and composition cues that do not conflict with Product Lock. Never copy a product, garment, accessory, prop, package, hand, model, logo, label, typography, watermark, or other branded element from the style reference.

Reference text is contamination, not a design instruction. Include typography only when the user supplies the exact text and explicitly asks for it.

## Canvas and composition

Preserve the source canvas and product scale by default. When a new aspect ratio is requested, expand or regenerate background around the locked product before considering a crop. Never stretch, squash, rescale disproportionately, or crop the product to force a ratio.

For ecommerce output, target approximately 15% clear padding around the product where the source and requested canvas make that possible without changing or shrinking the product. If it is not possible, report the constraint rather than violating Product Lock.

## Conflict rule

When a style, output profile, platform capability, or user request conflicts with a locked attribute, Product Lock wins. State the conflict and preserve the product.
